Claude Code CLI 实用命令指南：用好 /plan，让 AI 想清楚再动手

发表于2026-04-16分类于AI应用阅读次数

我们用 AI 写代码时通常会有这种烦恼：遇到一个需求，直接对着 AI 丢过去几句话，它不管三七二十一，瞬间劈里啪啦扔出几百行代码。

等你定睛一看，方向全错，不但原路退不回去，你还得给盲目的它收拾烂摊子。

怎么才能防止 AI "瞎写代码"？这时候，Claude Code CLI 命令行工具最精华的宝藏命令就派上用场了。

一、Claude Code 是什么？

一句话总结：它是 Anthropic 官方出品的 AI 编程命令行原生工具，是可以真正在你终端里读写项目文件的智能开发伙伴。

不同于网页版聊天的文本局限，它直接接管你的项目开发环境，并能主动帮你执行脚本和各类终端命令审核。

二、它能干嘛？实用命令大盘点

比起枯燥的盲猜开发流，它内置了几条能成倍提升生产力的核心指令。

1. /plan：让 AI "三思而后行"

这是整个 CLI 当中名副其实的“镇库之宝”！当你使用 /plan [你的需求] 时，它将进入绝对沉静的计划模式。

在这个模式下，它绝对不会急于输出任何代码，而是像一个缜密的资深架构师，严格地向你出具步骤分解方案。比如要改什么文件、定什么接口约定、分几个关键阶段。唯有你拍板同意后，它才会切换模式并开始动工。

2. /compact：卸下包袱的神器

聊着聊着，上百个回合过去后上下文马上被挤爆（导致响应变慢、它开始记混细节）。

你可以大呼一声 /compact 删除寒暄和走弯路的沟通，只保留登录模块的数据表建立逻辑。它刻意提纯信息，节约 Token，让你俩脑筋重新变灵光。

3. /clear：毫不留情的清脑丸

当你终于刚搞定一个庞杂的遗留 Bug，现在要全身心投入一个无关的新报表开发。一条 /clear 断舍离，斩断所有历史记录上下文，彻底防串台。

4. /bug：快速联络渠道

一旦触发 Anthropic 的核心抽风机制，直接在命令行里敲 /bug 提交带诊断信息的官方反馈，省时省力。

三、为什么这么厉害？

它颠覆了普通 AI 工具那种 "人类瞎猜意图式" 的赌博机制。

它不想完全替代人类的心智控制权，通过 /plan 的多方确认，它把终局拍板权还给了最适合统筹的人类。这种良性约束，让它摆脱出纯粹的"猴子打字机"状态。

四、怎么用？

完全零门槛（只要你有 Node 环境）：

  1. 全局安装利器： 
    npm install -g @anthropic-ai/claude-code
  2. 进入项目启动大门： 
    cd 你的项目代码目录/
claude
  3. 高光时刻触发： 
    /plan 这个项目需要接入支付宝支付，帮我规划下接口

五、实际体验：从"擦屁股"到真正的结对编程

我以前用网页版或是其他的 IDE 插件助手，总是提心吊胆，生怕没描述清细节导致它狂奔输出不能跑的神仙代码。但在 Claude Code 上我体验到了前所未有的舒心。

当你脑子里只有一个大概的功能雏形时敲下 /plan，他会实打实跟你一步步确认需求，然后才极其工整地给你提供执行计划，再也不会乱写代码！不但帮你把含糊的思路拨云见日，更免除了你事后在成堆 bug 废代码里捞人的灾难。这就是如假包换的高级生产力。

六、进阶/隐藏玩法

与其每次口头提醒规范，不如用契约精神约束它：在项目根目录创建一个 CLAUDE.md 的只读文件，写死你们的技术栈和铁律（例如：“任何业务抛异常前只用日志记录”）。哪怕你后续 /plan 闭口不谈规则，它照样倒背如流，规规矩矩办事。

七、总结

AI 的杀伤力不在于刷代码的字符数度，而在于精准无误。想从“修 bug 的苦工”中抽身，多动动嘴皮用 /plan ，让 AI 指挥自己。
