Claude Code 实战技术分享：让 AI 帮你写代码的正确姿势

发表于2026-04-11阅读次数

最近 Claude Code 突然成为了程序员圈里的顶流话题。和那些只会在聊天框里帮你回答问题的 AI 不一样，Claude Code 是一个真正能在你的终端里"干活"的 AI 编程工具——读代码、改代码、跑命令，它全包了。

作为一个已经用了一段时间的重度用户，今天来分享一些实战心得，帮助大家少走弯路。

一、Claude Code 是什么？

Claude Code 是 Anthropic 官方出品的 AI 编程命令行工具（CLI），核心能力是：

  • 🔍 理解你的整个项目代码库（不是只看一个文件）
  • ✏️ 直接读写文件，修改真实代码
  • 🖥️ 在终端执行命令，例如跑测试、安装依赖
  • 🌐 调用浏览器，抓取网页内容辅助编程
  • 🔁 多步骤自主完成任务，不是一问一答

简单来说，它不是一个"建议者"，而是一个真正动手干活的 AI 程序员助手。

二、安装与配置

1. 安装

前提：需要 Node.js 18+

npm install -g @anthropic-ai/claude-code

2. 配置 API Key

你需要一个 Claude 的 API Key，可以在 console.anthropic.com 创建。

export ANTHROPIC_API_KEY=sk-ant-xxxxxxxx

3. 启动

进入你的项目目录，直接运行：

cd your-project
claude

它会自动读取当前目录的所有代码文件作为上下文，然后进入交互式对话模式。

三、核心使用技巧

技巧 1：用自然语言描述任务，越具体越好

❌ 不好的 prompt：

帮我优化一下代码

✅ 好的 prompt：

帮我把 UserService.java 中的 getUserById 方法改成异步的，使用 CompletableFuture 包装，并添加异常处理逻辑，日志格式遵循项目现有的 Slf4j 风格

Claude Code 理解上下文的能力很强，但前提是你给够信息。把背景、约束、期望结果都说清楚，效果会成倍提升。

技巧 2：善用 /add 命令指定文件

默认情况下 Claude Code 会扫描整个项目，但如果项目很大，可以用 /add 精准指定要处理的文件：

/add src/main/service/OrderService.java
/add src/main/repository/OrderRepository.java

这样它只会聚焦在这几个文件上操作，不会误改其他无关代码。

技巧 3：配合 Git 使用，安全感拉满

Claude Code 会直接修改你的真实文件，所以 强烈建议在干净的 Git 分支上操作：

git checkout -b feature/ai-refactor
claude  # 开始 AI 操作
git diff  # 看看它改了什么
git add . && git commit -m "AI assisted refactor"

这样即使 AI 改出问题，一个 git checkout . 就能全部撤销，完全无风险。

技巧 4：让它写测试，不只是写代码

很多人只让 Claude Code 写业务代码，其实它在写单元测试方面表现非常出色。直接说：

为 PaymentService 中的 processRefund 方法写 JUnit 5 单元测试，覆盖正常退款、余额不足、订单不存在三种场景，使用 Mockito mock 依赖

它写出来的测试代码通常直接可用，命名清晰，覆盖率也不低。

技巧 5：Code Review 模式

让 Claude Code 帮你做代码 Review，发现潜在问题：

Review 一下 src/controller 目录下的所有 Controller，检查是否有未处理的异常、缺失的参数校验、可能的 SQL 注入风险，给出改进建议

四、进阶玩法：CLAUDE.md 自定义规范文件

这是很多人不知道的杀手锏功能。在项目根目录新建一个 CLAUDE.md 文件，写入你的项目规范，Claude Code 每次启动都会自动读取：

# 项目规范

## 技术栈
- Java 17 + Spring Boot 3.x
- MyBatis-Plus 作为 ORM
- Redis 做缓存，统一使用 RedisTemplate

## 代码规范
- 所有 Service 方法必须加 @Transactional 注解
- 日志统一用 Slf4j，禁止使用 System.out.println
- 异常统一抛出 BusinessException，不允许 catch 后吞掉

## 命名规范
- 接口以 I 开头，例如 IUserService
- VO 类放在 vo 包下，DTO 放在 dto 包下

有了这个文件，你不需要每次都反复交代规范，Claude Code 会自动遵守。这一步能节省你 80% 的重复沟通成本。

五、常见坑与注意事项

⚠️ 坑 1：它有时候会过度修改

Claude Code 比较"热情"，有时候你只让它改一个方法，它可能顺手把周边代码也重构了。建议每次任务结束后用 git diff 仔细检查，确认改动范围符合预期。

⚠️ 坑 2：复杂业务逻辑还是需要人工把关

AI 写的代码逻辑层面大多正确，但涉及到你们系统特有的业务规则（比如特殊的状态机、复杂的权限逻辑），它容易"凭想象"写，需要你仔细核对。

⚠️ 坑 3：费用控制

Claude Code 按 token 消耗计费，大项目扫描上下文较多，成本会比较高。建议用 /add 精确指定文件范围，避免不必要的 token 浪费。

六、总结

Claude Code 代表了 AI 辅助编程的一个新范式——从"问答式"到"代理式"。它不只是告诉你答案，而是真的替你去做。

用好它的关键在于：

  1. Prompt 要具体，背景、约束、期望都说清楚
  2. 配置好 CLAUDE.md，一次规范，终身受益
  3. 配合 Git 使用，大胆用、放心改
  4. 人工 Review 不能省，AI 是助手不是替代品
